# Edge AI Smart Glove — ESP32 Firmware

On-device TinyML gesture recognition. The firmware samples an MPU6050 IMU,
extracts the same 30 statistical features used during training, and runs the
INT8-quantized TensorFlow Lite model directly on the ESP32 with TensorFlow
Lite Micro — no cloud or host PC involved at inference time.

## Hardware

| Component | Notes |
|-----------|-------|
| ESP32 dev board | Any board supported by `board = esp32dev` |
| MPU6050 IMU | Accelerometer + gyroscope, I2C |

### Wiring

| MPU6050 | ESP32 |
|---------|-------|
| VCC     | 3.3V  |
| GND     | GND   |
| SCL     | GPIO 22 (default I2C SCL) |
| SDA     | GPIO 21 (default I2C SDA) |

## Pipeline

```
MPU6050 (50 Hz)
      │  50-sample window (1 second)
      ▼
Feature Extraction (30 features)
  mean / std / min / max / rms
  per axis: ax, ay, az, gx, gy, gz
      │
      ▼
TensorFlow Lite Micro (INT8)
      │
      ▼
Predicted Gesture + Confidence (Serial)
```

Gesture classes: `flick_down`, `flick_up`, `idle`, `wave_left`, `wave_right`.

## Source layout

| File | Purpose |
|------|---------|
| `include/model_config.h` | Auto-generated scaler/label constants (see below) |
| `include/imu.h` / `src/imu.cpp` | Raw I2C driver for the MPU6050 |
| `include/feature_extraction.h` / `src/feature_extraction.cpp` | Reproduces `src/preprocessing.py` + `src/feature_extraction.py` on-device |
| `include/predictor.h` / `src/predictor.cpp` | TensorFlow Lite Micro inference (quantize → invoke → dequantize) |
| `include/model_data.h` / `src/model_data.cpp` | Embedded INT8 `.tflite` model bytes |
| `src/main.cpp` | Sampling loop that ties it all together |

## Keeping the model in sync

`model_data.cpp` and `model_config.h` are both generated by
[`convert_model.py`](../convert_model.py) from the trained artifacts in
`models/trained/` (`gesture_model_int8.tflite`, `scaler.pkl`,
`label_encoder.pkl`). After retraining, regenerate both from the project root:

```bash
python convert_model.py
```

Do not hand-edit `model_data.cpp` or `model_config.h`.

## Build & upload (PlatformIO)

```bash
pio run                # build
pio run -t upload      # flash
pio device monitor      # view predictions at 115200 baud
```

## Notes

- The 50 Hz / 50-sample (1 second) window matches how
  `data/public/gesture_dataset/gesture_dataset.csv` was recorded.
- The accelerometer/gyroscope full-scale ranges (±2g / ±250 deg/s) match the
  value ranges observed in the training dataset.
